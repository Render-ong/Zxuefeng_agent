# 数据字典

> 本文档描述 XF_agent 后端所有数据库表结构、向量存储和配置文件。
> 修改表结构或添加新字段时必须同步更新此文档。

---

## 1. admission_clean.db（只读，录取数据库）

路径：`GraphRAG/admission_clean.db`

### 1.1 admission 表（260,884 行）

各省份各年份各高校各专业的录取数据。

| 列名 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | INTEGER PK | 自增主键 | `1` |
| `province` | TEXT | 省份 | `"浙江"` |
| `year` | INTEGER | 年份 | `2024` |
| `category` | TEXT | 科类 | `"综合"` / `"物理"` / `"历史"` |
| `batch` | TEXT | 批次 | `"本科"` / `"专科"` |
| `school_name` | TEXT | 学校名称 | `"浙江大学"` |
| `major_name` | TEXT | 专业名称 | `"计算机科学与技术"` |
| `score` | INTEGER | 最低录取分数 | `670` |
| `rank` | INTEGER | 最低录取位次 | `3000` |
| `quota` | INTEGER | 招生计划数 | `50` |
| `source_file` | TEXT | 数据来源文件 | `"zhejiang_2024.xlsx"` |

### 1.2 batch_lines 表（0 行，预留给批次线数据）

| 列名 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `province` | TEXT | 省份 |
| `year` | INTEGER | 年份 |
| `category` | TEXT | 科类 |
| `batch` | TEXT | 批次 |
| `score` | INTEGER | 批次线分数 |

## 2. user_data.db（用户会话 + 全局画像）

路径：`data/user/user_data.db`
管理模块：`GraphRAG/api_server.py`

### 2.1 user_profiles 表

全局用户画像（按 user_id 隔离，跨对话共享）。

| 列名 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `user_id` | TEXT PK | 用户 ID（微信 openid 或 dev_user） | `"oXXX..."` |
| `name` | TEXT | 姓名 | `"张雪儿"` |
| `province` | TEXT | 省份 | `"安徽"` |
| `score` | TEXT | 分数 | `"699"` |
| `rank` | TEXT | 位次 | `"2"` |
| `customProfile` | TEXT | 自定义描述 | `"无所谓"` |
| `updated_at` | TEXT | 更新时间（ISO 8601） | `"2026-06-26T03:52:08"` |

### 2.2 sessions 表

登录会话（token → user_id 映射）。

| 列名 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `token` | TEXT PK | session token（UUID hex） | `"11eae58d0ba6..."` |
| `user_id` | TEXT | 用户 ID | `"dev_user"` |
| `created_at` | TEXT | 创建时间 | `"2026-06-26T12:37:47"` |
| `expires_at` | TEXT | 过期时间（7天后） | `"2026-07-03T12:37:47"` |

索引：`idx_sessions_user ON sessions(user_id)`

## 3. langgraph_data.db（对话 + 消息 + 画像）

路径：`data/user/langgraph_data.db`
管理模块：`langgraph/database.py` + `langgraph/user_manager.py`

### 3.1 users 表

| 列名 | 类型 | 说明 |
|------|------|------|
| `user_id` | TEXT PK | 用户 ID |
| `nickname` | TEXT | 昵称 |
| `created_at` | TEXT | 创建时间 |
| `updated_at` | TEXT | 更新时间 |

### 3.2 conversations 表

| 列名 | 类型 | 说明 |
|------|------|------|
| `conversation_id` | TEXT PK | 对话 ID（UUID） |
| `user_id` | TEXT | 创建者 ID（外键 → users） |
| `title` | TEXT | 对话标题 |
| `created_at` | TEXT | 创建时间 |
| `updated_at` | TEXT | 更新时间 |

索引：`idx_conversations_user ON conversations(user_id)`

### 3.3 messages 表

| 列名 | 类型 | 说明 |
|------|------|------|
| `message_id` | TEXT PK | 消息 ID（UUID） |
| `conversation_id` | TEXT | 所属对话（外键 → conversations） |
| `user_id` | TEXT | 所属用户 |
| `role` | TEXT | 角色：`"user"` / `"assistant"` |
| `content` | TEXT | 消息内容 |
| `created_at` | TEXT | 创建时间 |

索引：
- `idx_messages_conv ON messages(conversation_id)`
- `idx_messages_user ON messages(user_id)`

### 3.4 profiles 表

Per-conversation 画像（与 user_data.db 的全局画像不同，这是每个对话独立的画像快照）。

| 列名 | 类型 | 说明 |
|------|------|------|
| `user_id` | TEXT | 用户 ID（联合主键） |
| `conversation_id` | TEXT | 对话 ID（联合主键） |
| `profile_data` | TEXT | JSON 格式的画像数据 |
| `updated_at` | TEXT | 更新时间 |

主键：`(user_id, conversation_id)`
索引：`idx_profiles_user ON profiles(user_id)`

## 4. checkpoints.db（LangGraph 状态快照）

路径：`data/user/checkpoints.db`
管理模块：`langgraph/agent.py`（通过 `langgraph-checkpoint-sqlite`）

由 LangGraph 框架自动管理，存储 AgentState 的历史快照。
表结构由 `langgraph-checkpoint-sqlite` 包定义，通常包含 `checkpoints`、`writes` 等表。

**作用**：多轮对话时自动恢复 `state.profile` 和 `state.messages`。
**thread_id 格式**：`{user_id}:{conversation_id}`

## 5. 向量存储

路径：`GraphRAG/storage/vector/`

### 5.1 majors.npz

专业名向量索引。

- **向量文件**：`majors.npz`（numpy 压缩格式）
- **元数据**：`meta.json`
  ```json
  {
    "model": "BAAI/bge-small-zh-v1.5",
    "dimension": 512,
    "count": 5000,
    "build_time": "2026-06-20T10:00:00"
  }
  ```
- **用途**：关键词扩展（"想学AI" → 相关专业列表）
- **构建脚本**：`GraphRAG/scripts/build_vectors.py`

### 5.2 schools.npz

学校名向量索引。

- **向量文件**：`schools.npz`
- **元数据**：`schools_meta.json`
- **用途**：校名消歧、语义匹配
- **构建脚本**：`GraphRAG/scripts/build_school_index.py`

### 5.3 knowledge.npz

知识 chunk 向量索引（张雪峰知识库）。

- **向量文件**：`knowledge.npz`
- **元数据**：`knowledge_meta.json`
- **来源**：`GraphRAG/data/knowledge/chunks.json`
- **用途**：方法论 RAG（GraphRAG 节点检索）
- **构建脚本**：`GraphRAG/scripts/build_knowledge_index.py`

## 8. 日志

路径：`logs/app.log`
管理模块：`log_setup.py`

- 格式：`%(asctime)s [%(levelname)s] %(name)s | %(message)s`
- 轮转：10MB × 5 份（`app.log`, `app.log.1`, ...）
- 级别：`XF_LOG_LEVEL` 环境变量控制（默认 `INFO`）
- 控制台：默认输出到 stderr，`XF_LOG_CONSOLE=0` 关闭
