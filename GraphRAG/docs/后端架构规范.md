## 2. 目录职责

| 目录 | 职责 | 入口文件 |

## 3. 请求数据流

### 3.1 对话请求 (`POST /api/chat`)

```
1. 前端发送 { message, conversation_id, user_profile, api_config }
                │
2. api_server.py 鉴权 (Bearer token → sessions 表反查 user_id)
                │
3. 画像注入：前端 user_profile 合并到 user_data.db.user_profiles
   拼接 profile_hint 到 enhanced_message
                │
4. lg_agent.run(enhanced_message, user_id, conversation_id)
                │
5. analyze 节点：从消息抽取 province/score/rank/majors 等 → 更新 profile
                │
6. 路由：
   needs_ask? → ask 节点 → END（追问，等下一轮）
   intent=recommend? → recommend 节点（SQL 冲稳保）
   intent=compare?   → compare 节点
   intent=explain?   → explain 节点
   intent=policy?    → policy 节点
   其他?             → 直接 graphrag
                │
7. graphrag 节点：向量检索 + 图谱上下文
                │
8. web_search 节点：联网搜索（可选，需 tavily key）
                │
9. generate 节点：调用 LLM API 生成回复
                │
10. 返回 { reply, debug, profile, needs_ask, ask_questions, conversation_id }
```

### 3.2 登录请求 (`POST /api/auth/login`)

```
前端 wx.login() 拿 code
    → api_server.py
    → 微信 code2session API（有 wx_appid 时）或 dev_user 回退
    → 签发 UUID token → 写入 sessions 表（TTL 7天）
    → 返回 { user_id, token }
```

## 4. 三数据库架构

| 数据库 | 路径 | 读写 | 内容 |
|--------|------|------|------|
| `admission_clean.db` | `GraphRAG/admission_clean.db` | 只读 | 录取数据（26万行）、批次线 |
| `user_data.db` | `data/user/user_data.db` | 读写 | 用户 session、全局画像 |
| `langgraph_data.db` | `data/user/langgraph_data.db` | 读写 | 用户、对话、消息、per-conversation 画像 |
| `checkpoints.db` | `data/user/checkpoints.db` | 读写 | LangGraph 状态快照（多轮上下文） |

**设计原则**：
- 公共数据（录取库）与用户数据物理隔离
- 用户数据集中存放 `data/user/`，便于备份/迁移
- 所有 SQLite 连接启用 WAL + busy_timeout=5000，缓解并发锁

## 5. 配置体系

### 5.1 配置文件

| 文件 | 位置 | 用途 |
|------|------|------|
| `config.json` | `GraphRAG/` | LLM API Key/URL/Model、Tavily Key、微信 AppID/Secret |
| `api_config.json` | `langgraph/` | 前端设置页写入的 LLM 配置（运行时生成） |
| `retrieval.yaml` | `GraphRAG/config/` | 向量检索参数、GraphRAG 阈值、Ollama 配置 |
| `rank_bands.yaml` | `GraphRAG/config/` | 冲稳保位次带宽、每档学校/专业数量限制 |

### 5.2 配置优先级

```
环境变量 > api_config.json (前端写入) > config.json > 内置默认值
```

**LLM 配置**（`langgraph/llm_config.py`）：
- `LLM_URL` / `LLM_KEY` / `LLM_MODEL` 环境变量
- `langgraph/api_config.json`（前端设置页写入）
- `GraphRAG/config.json`
- 内置默认：`deepseek-v4-flash` @ `https://api.deepseek.com`

**微信配置**（`GraphRAG/api_server.py`）：
- `config.json` 中的 `wx_appid` + `wx_secret`
- 缺失时需设 `XF_ALLOW_DEV=1`（仅本地开发）

### 5.3 环境变量清单

| 变量 | 用途 | 默认 |
|------|------|------|
| `XF_ALLOW_DEV` | 允许无微信配置启动（`dev_user` 回退） | 未设置 |
| `XF_LOG_LEVEL` | 日志级别 | `INFO` |
| `XF_LOG_CONSOLE` | 是否输出到控制台（`0` 关闭） | `1` |
| `LLM_URL` | LLM API 地址 | 空（用 config.json） |
| `LLM_KEY` | LLM API Key | 空 |
| `LLM_MODEL` | LLM 模型名 | 空 |
| `DEEPSEEK_KEY` | DeepSeek API Key（`LLM_KEY` 的别名） | 空 |
| `TAVILY_KEY` | Tavily 联网搜索 Key | 空 |

## 6. 向量存储

存储位置：`GraphRAG/storage/vector/`

| 文件 | 内容 | 用途 |
|------|------|------|
| `majors.npz` + `meta.json` | 专业名向量索引 | 关键词扩展（"想学AI" → 相关专业） |
| `schools.npz` + `schools_meta.json` | 学校名向量索引 | 校名消歧、语义匹配 |
| `knowledge.npz` + `knowledge_meta.json` | 知识 chunk 向量 | 方法论 RAG |
| `meta.json` | 索引元数据 | 模型名、维度、构建时间 |

模型：`BAAI/bge-small-zh-v1.5`（轻量中文，8GB 显存友好）

## 7. 模块依赖关系

```
api_server.py
  ├── langgraph/agent.py (通过 importlib 加载，避免包名冲突)
  │     ├── langgraph/nodes/analyze.py → GraphRAG/engine/extractor.py
  │     ├── langgraph/nodes/recommend.py → GraphRAG/engine/recommend.py
  │     ├── langgraph/nodes/graphrag.py → GraphRAG/engine/graphrag.py
  │     ├── langgraph/nodes/web_search.py → GraphRAG/engine/web_search.py
  │     ├── langgraph/nodes/generate.py → langgraph/llm_config.py
  │     ├── langgraph/database.py
  │     └── langgraph/user_manager.py
  ├── langgraph/llm_config.py
  └── log_setup.py

GraphRAG/engine/recommend.py (编排入口)
  ├── engine/sql_recommend.py
  ├── engine/vector_store.py
  └── engine/graph_store.py

GraphRAG/engine/graphrag.py
  ├── engine/vector_store.py (社区向量检索)
  └── engine/graph_store.py (图遍历)
```

**关键约束**：
- `langgraph/` 目录与 pip 安装的 `langgraph` 包同名，`api_server.py` 必须用 `importlib` 加载 `agent.py`
- `langgraph/nodes/*.py` 通过 `sys.path.insert` 引用 `GraphRAG/engine/`
- 所有模块共用 `log_setup.py` 的日志配置

## 8. 安全约束

| 约束 | 实现 |
|------|------|
| 鉴权 | Bearer token → sessions 表反查 user_id，TTL 7天 |
| 防 IDOR | 所有对话/画像操作校验 user_id 归属 |
| 密钥不落盘 | LangGraph state 中不放 api_config，避免随 checkpoint 持久化 |
| 生产环境 | 必须配置 wx_appid/wx_secret，禁用 XF_ALLOW_DEV |
| 输入校验 | message 长度 ≤ 4000 字符 |
